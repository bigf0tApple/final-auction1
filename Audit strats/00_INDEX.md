# Audit strats — Index

Use these documents as reusable audit playbooks. They are designed to be copy/paste friendly.

## What to use when

- If you’re auditing a page or feature end-to-end: start with `03_FEATURE_FLOW_STRATEGY.md`.
- If you’re chasing “what happens when I click this”: use `01_BUTTON_AUDIT_STRATEGY.md`.
- If you’re mapping component interconnections: use `02_COMPONENT_CONNECTIONS_STRATEGY.md`.
- If you need a repeatable regression pass before shipping: use `04_REGRESSION_MATRIX_TEMPLATE.md`.
